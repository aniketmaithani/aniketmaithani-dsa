Fast decision table

Before coding, ask: “What is changing when I move through the array?”

Problem signal Use this pattern Why?
Many range sum queries, sum from index L to R, repeated subarray sum checks Prefix Sum Precompute running totals once, then answer range sums quickly.

The beginner rule

1. Is it contiguous?If the problem says subarray or substring, think Kadane or Sliding Window first.
2. Is size fixed?If size k is fixed, use fixed sliding window.
3. Is size flexible?If asked longest/shortest with condition, use variable sliding window.
4. Is it range query?If repeated sum from L to R is needed, use prefix sum.
5. Is array sorted?If yes and pair/triplet is needed, try two pointers.

1. Prefix Sum

Range sumSubarray sumPrecomputation

Use when: You need sum of elements between two indexes again and again, or you need to detect whether some previous prefix can help form a target sum.

  • Best signal: “sum from left to right”, “range query”, “subarray sum equals k”.
  • Brute force: Sum every range again, often O(n²) or O(q × n).
  • Optimized: Build prefix once in O(n), then answer range sum in O(1).
  • Common trap: For subarray sum equals k with negative numbers, normal sliding window may fail. Prefix sum + hashmap is safer.
Python template
prefix = [0]
for num in nums:
    prefix.append(prefix[-1] + num)

# sum from left to right inclusive
range_sum = prefix[right + 1] - prefix[left]

2. Kadane's Algorithm

Maximum subarrayContiguousNegative numbers

Use when: You need the best contiguous subarray sum/product and each index asks: should I extend previous subarray or start fresh?

  • Best signal: “maximum subarray”, “largest sum contiguous subarray”.
  • Core thought: A negative past can hurt the future, so drop it.
  • Time complexity: O(n), because each element is processed once.
  • Common trap: Initialize with nums[0], not 0, otherwise all-negative arrays break.
Python template
best = nums[0]
current = nums[0]

for i in range(1, len(nums)):
    current = max(nums[i], current + nums[i])
    best = max(best, current)

return best

3. Two Pointers

Sorted arrayPairsOpposite ends

Use when: Two positions move through the array/string and each movement removes useless possibilities.

  • Best signal: Sorted array, pair sum, palindrome, left/right boundary.
  • Typical movement: If sum is too small, move left. If sum is too large, move right.
  • Time complexity: Usually O(n), because every pointer moves at most n times.
  • Common trap: Do not use two pointers blindly on unsorted pair-sum problems unless sorting is allowed.
Python template for sorted pair sum
left, right = 0, len(nums) - 1

while left < right:
    total = nums[left] + nums[right]
    if total == target:
        return [left, right]
    elif total < target:
        left += 1
    else:
        right -= 1

4. Fixed Sliding Window

Window size kMax averageConstant size

Use when: The problem clearly says window/subarray/substring of exactly size k.

  • Best signal: “size k”, “length k”, “exactly k elements”.
  • Core thought: Compute first window, then slide by removing outgoing and adding incoming.
  • Time complexity: O(n). Without it, checking every window from scratch costs O(n × k).
  • Common trap: Main loop starts from index k, because first k elements are already included.
Python template
window_sum = sum(nums[:k])
best = window_sum

for i in range(k, len(nums)):
    window_sum = window_sum - nums[i - k] + nums[i]
    best = max(best, window_sum)

return best

5. Variable Sliding Window

LongestShortestCondition based

Use when: Window size is not fixed. You expand and shrink until the condition is valid or invalid.

  • Best signal: “longest substring”, “minimum length subarray”, “at most k”, “at least target”.
  • Core thought: Right pointer expands. Left pointer shrinks when the condition demands it.
  • Time complexity: O(n), because each pointer moves forward only.
  • Common trap: This works best when the condition changes predictably as the window grows/shrinks.
Python template
left = 0
answer = 0
state = {}

for right in range(len(arr)):
    # add arr[right] to state

    while condition_is_broken:
        # remove arr[left] from state
        left += 1

    answer = max(answer, right - left + 1)

Confusion: Two Pointers vs Sliding Window

Both use left and right indexes, so beginners confuse them. The difference is the purpose.

  • Two pointers: Usually two independent pointers moving to eliminate search space. Example: sorted pair sum.
  • Sliding window: Left and right form a continuous window. The window content matters.
  • Simple test: If the answer depends on everything between left and right, it is probably sliding window.
  • Another test: If you only care about values at left and right, it is probably two pointers.
Mental model
Two Pointers:
left value + right value decides movement

Sliding Window:
all elements between left and right form the current state

Final revision cheat sheet

Prefix Sum

Use for range sums and subarray sum counting. Great when previous accumulated sum helps.

Kadane

Use for best contiguous subarray. At every index, decide extend or restart.

Two Pointers

Use when moving two indexes removes impossible answers, especially sorted arrays.

Fixed Window

Use when window length is exactly k. Remove left, add right.

Variable Window

Use for longest/shortest window satisfying a condition.
